### Key findings

1. **Zero-orders root cause confirmed**: `"Strong Buy".upper()` =
   `"STRONG BUY"` (space) is NOT in `_BUY_RECS = {"BUY", "STRONG_BUY"}`
   (underscore). All "Strong Buy" and "Strong Sell" recommendations
   from Gemini were silently dropped for 27+ days.

2. **optimizer_best.json warm-start corruption**: The `_load_previous_best`
   method copies params from `_PARAM_BOUNDS` and `_CATEGORICAL_PARAMS`
   keys only (line 662-664). If a param like `fm_weight` is in the
   strategy_params but not in the bounds dict keys, it's silently
   dropped. Combined with `_save_best_params()` persisting the
   potentially incomplete `self.best_params`, the corruption propagates.

3. **Two-factor Sharpe degradation**:
   - Factor 1 (fixed): Wrong params in `optimizer_best.json`
   - Factor 2 (unfixed): Data or code drift causing same params to
     produce different results between March and April runs

4. **BQ HDW tables are empty**: `Market_L1_Snapshots` and
   `Fundamental_PIT_Data` both have 0 rows. The backtest engine's
   `historical_prices` table was not found in any BQ dataset.
   Backtest data source needs investigation.
