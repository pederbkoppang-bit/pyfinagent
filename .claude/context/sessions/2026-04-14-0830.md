Key research findings that drove design:

3. **Drawdown should be computed on mark-to-market equity**, not daily closes — the kill switch needs to fire intraday.
4. **5/10/15 tier ladder** is the industry convention across QuantConnect, Robot Wealth, QuantVPS, QuantifiedStrategies.

**`backend/agents/mcp_servers/signals_server.py` (+336 / -13):**

1. `__init__`: added `self._peak_equity: Optional[float] = None` (2 lines + comment).
2. `get_risk_constraints()`: extended dict with 6 new keys (`max_position_pct=5.0`, `max_position_usd=1000.0`, `stop_loss_pct=8.0`, `trail_stop_pct=3.0`, `drawdown_warning_pct=-5.0`, `drawdown_derisk_pct=-10.0`). Existing 5 keys unchanged.
3. `size_position(signal, portfolio) -> float`: hybrid lite-formula. Always-included hard percent-cap arm (`min(equity*max_pct, max_usd)`). Optional half-Kelly arm (`0.5 * confidence * equity`) — skipped if confidence is None/invalid. Optional inverse-vol arm (`target_vol_pct/100 * equity / annualized_vol`) — skipped if ann_vol <= 0. Returns `min()` of included candidates. Explicit `signal.size_usd` bypasses the formula. Returns 0.0 on any degraded path. Never raises.
4. `check_stop_loss(portfolio) -> list[dict]`: iterates `portfolio["positions"]`, for each position computes `(current-entry)/entry` (fixed stop) and `(current-peak)/peak` (trailing stop). Returns triggered positions with reason + loss_pct. Malformed positions skipped silently. Pure, no mutation. Default peak = `max(entry, current)` when missing.
5. `track_drawdown(portfolio) -> dict`: lazy-init `_peak_equity` on first call. Updates peak on new highs. Computes `dd_pct = ((equity-peak)/peak)*100`. Classifies into ok/warning/derisk/kill via 5/10/15 ladder from constraints. Returns `{peak, equity, drawdown_pct, tier, kill_switch}`.
6. `publish_signal` step 5: replaced the 10-line inline v1 sizing block with `amount_usd = self.size_position(signal, portfolio)`. Steps 1-4 and 6-9 byte-identical to origin/main. The `size_usd` override path now lives inside `size_position` itself.
