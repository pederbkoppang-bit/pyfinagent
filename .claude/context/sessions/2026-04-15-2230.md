Prior session (1838) explicitly flagged Phase 4.4.4.3 as the next
tractable item on its "next run should" list, with the caveat that
it would be a hard block if the stop-loss exit logic was not
present in `paper_trader.py`. On inspection:

- `paper_trader.check_stop_losses()` (lines 282-291) returns a list
  of triggered tickers but does NOT emit orders -- it's a read-only
  helper.
- The canonical stop-loss SELL emission lives in
  `portfolio_manager.decide_trades()` at lines 73-85:

  ```python
  stop = pos.get("stop_loss_price")
  current = pos.get("current_price", 0)
  if stop and current and current <= stop:
      orders.append(TradeOrder(
          ticker=ticker, action="SELL", reason="stop_loss",
          price=current,
      ))
      continue
  ```

- The `continue` at line 85 is load-bearing: it skips the downstream
  signal-classification block that would otherwise also emit a
  `sell_signal` / `signal_downgrade` order for the same ticker.
  This pins that the stop-loss path takes precedence over any
  concurrent re-evaluation signal.
- `autonomous_loop.py` consumes the TradeOrders from `decide_trades`
  and calls `paper_trader.execute_sell(ticker, reason="stop_loss",
  price=...)`. So the stop-loss path IS present and is the canonical
  exit route for the paper trading loop.

With the exit logic confirmed, 4.4.4.3 became tractable as a pure
verification drill -- no code gate needed. A drill mirroring the
