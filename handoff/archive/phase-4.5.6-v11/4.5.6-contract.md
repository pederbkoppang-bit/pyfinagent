# Step 4.5.6 Contract -- Live intraday prices + chart refresh (no-cycle path)

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`
**Research tier:** simple (phase-level contract already cites yfinance rate-limit "cache + rate-gate")

## Hypothesis
A server-side TTL cache + rate-gate on yfinance plus a visibility-aware 60s frontend poll gives operators fresher-than-cycle prices for open positions without exceeding Yahoo's 360-req/hr ceiling. Including `age_sec` in the response prevents the documented "polling without staleness indicator" anti-pattern.

## Success criteria (from masterplan 4.5.6)
1. endpoint_GET_paper_trading_live_prices_exists -- `GET /api/paper-trading/live-prices?tickers=AAPL,MSFT` returns 200.
2. frontend_polls_every_60s_only_when_tab_visible -- `useLivePrices` uses `document.hidden` + `visibilitychange` per MDN canonical pattern; 60s `setInterval`; clears interval on unmount and on tab hide.
3. yfinance_rate_limit_respected -- process-wide refresh cap of 30/min (well under Yahoo's 360/hr and 2024-updated burst limits); per-ticker 60s TTL; `rate_gated` flag returned when the gate trips.
4. evaluator_critique_pass.

## Research findings (moderate summary, full entry appended to RESEARCH.md by researcher subagent)

- yfinance rate ceiling is 360/hr documented; burst detection tightened November 2024 (GitHub issue #2128). 1m interval capped at 7-day lookback.
- 60s TTL is the defensible midpoint for cycle-based strategies (Moldstud, AWS caching best practices). 30s halves the quota with no signal benefit; 300s risks showing stale prices right after a new entry.
- Page Visibility API (`document.hidden` + `visibilitychange`) is 2026 canonical; MDN baseline-widely-available; no React first-party hook.
- 5-consecutive-failure threshold is the cross-ecosystem modal (Resilience4j, Polly defaults).
- Anti-pattern: polling without staleness indicator (Coinpaprika 2024) -- every response must carry `age_sec` so the UI can render how old the price is.

## Design

**Backend (`backend/services/live_prices.py`):**
- `LivePriceCache` class, module-level singleton via `get_live_cache()`.
- `get_many(tickers)` returns `{ticker: {price, age_sec, cached, rate_gated?}}`.
- Thread-safe via `threading.Lock` (same pattern as `api_cache.py`, `perf_tracker.py`).
- Per-ticker 60s TTL; process-wide rate gate: `MAX_UNIQUE_REFRESH_PER_MIN = 30`.
- `_fetch_price` uses `yf.Ticker(t).history(period="1d", interval="1m")` and takes the last close.
- Ticker input sanitized (alnum + `.` + `-` only).

**Endpoint (`backend/api/paper_trading.py`):**
- `GET /api/paper-trading/live-prices?tickers=AAPL,MSFT`.
- Cap: 50 tickers per request (HTTP 400 otherwise).
- Returns `{prices, ttl_sec: 60, count}`.
- `await asyncio.to_thread(cache.get_many, ...)` to keep the event loop unblocked.

**Frontend (`frontend/src/lib/useLivePrices.ts`):**
- Hook `useLivePrices(tickers, enabled)` returns `{prices, error, updatedAt}`.
- On mount / tickers change: fire once, then `setInterval(tick, 60_000)`.
- `document.addEventListener("visibilitychange", onVis)` -- if `!document.hidden`, tick immediately to give the user fresh data when they switch back.
- Inside `tick()`: `if (document.hidden) return;` -- skip the fetch when hidden.
- 5-consecutive-failure counter, surfaces `error` only after the 5th.
- Returns entries of shape `{price, age_sec, cached, rate_gated?}`.

**Integration (`frontend/src/app/paper-trading/page.tsx`):**
- Positions tab `Current` column now shows the live price with an age annotation (`$150.32 (12s)`); falls back to `pos.current_price` when live is unavailable.
- Hook `enabled={tab === "positions" && positions.length > 0}` -- does not poll on Trades/Chart/Reality-gap tabs.

## Anti-patterns guarded
- **Polling without staleness** -- UI shows `age_sec` per ticker.
- **Polling while tab hidden** -- `document.hidden` early return.
- **Unbounded yfinance calls** -- 30/min process-wide rate gate; 60s per-ticker TTL.
- **Silent failures** -- 5-in-a-row counter surfaces `error` to the caller.

## Out of scope
- Server-sent events / WebSocket streaming (paid feeds / different architecture).
- Sub-minute freshness (strategy is cycle-based).
- Paid data-vendor integration (phase-4.4 Go-Live territory).

## Risk
- yfinance is a community-maintained client; sudden Yahoo endpoint changes could break the cache. Fallback: cache serves stale entries with `cached: true`, frontend shows `age_sec` so operators can tell.
- Tab visibility differs on mobile Safari (pagehide/pageshow); out-of-scope for this step -- the primary target is desktop operator dashboards.
