# Step 4.5.4 Contract -- Go-Live Gate checklist widget

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`

## Hypothesis
Five deterministic booleans -- visible in the page header, computed server-side -- are sufficient to gate live-capital promotion. They force the decision to rest on measurable evidence (trades, PSR, DSR, reality gap, drawdown) rather than operator intuition.

## Success criteria (from masterplan 4.5.4)
1. gate_booleans_include_trades_ge_100
2. gate_booleans_include_psr_ge_0_95_sustained_30d
3. gate_booleans_include_dsr_ge_0_95
4. gate_booleans_include_sr_gap_le_30pct
5. gate_booleans_include_max_dd_within_tolerance
6. promote_to_live_button_disabled_unless_all_green
7. evaluator_critique_pass

## Design
- `backend/services/paper_go_live_gate.py` (new): `compute_gate()` reuses paper_metrics_v2 (PSR/DSR/n_obs), paper_round_trips (trade count), and reconciliation (SR-gap proxy via divergence). Returns `booleans`, `promote_eligible` (= AND), `details`, `thresholds`, `computed_at`.
- `backend/api/paper_trading.py`: new `GET /gate` endpoint, 120s cache.
- `frontend/src/components/GoLiveGateWidget.tsx` (new): 5 pass/fail cards + aggregate badge + Promote button. Disabled unless `promote_eligible === true`.
- `frontend/src/app/paper-trading/page.tsx`: widget placed above summary cards; eager-loaded on page mount.
- `frontend/src/lib/icons.ts`: +IconCheckCircle, IconXCircle, IconInfo aliases.
- `frontend/src/lib/api.ts`: +getPaperGate wrapper.

## Thresholds (fixed constants; not user-tunable this phase)
- TRADES_THRESHOLD = 100
- PSR_SUSTAINED_DAYS = 30 (n_obs >= 30 AND PSR >= 0.95)
- DSR_THRESHOLD = 0.95
- SR_GAP_THRESHOLD = 0.30 (30% relative; proxied by reconciliation latest_divergence_pct)
- MAX_DD_ABS_TOLERANCE = 20.0% (absolute cap on realized max drawdown)

## Anti-patterns guarded
- Button disabled (not hidden) -- UX needs the affordance visible so the operator sees *why* they can't promote.
- booleans are strict booleans, not truthy numbers -- prevents JS coercion bugs.
- All thresholds returned in response so the UI doesn't hardcode them -- single source of truth.

## Out of scope
- Live brokerage hookup for the Promote button (that's phase-4.4 Go-Live Checklist).
- Sustained-PSR computation over a true 30-day rolling window (current implementation uses `n_obs >= 30` + `psr >= 0.95`; strict per-day rolling aggregation can come with the metrics_v2 BQ table populated).
- Operator-configurable thresholds.

## Risk
- Reconciliation endpoint dependency: if yfinance is down, the SR-gap check will likely fail (conservative) -- acceptable.
- Gate widget eager-loads on every mount; 120s cache prevents BQ hammering.
