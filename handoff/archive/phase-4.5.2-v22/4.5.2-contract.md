# Step 4.5.2 Contract — Round-trip performance metrics

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`

## Hypothesis
Adding FIFO-paired round-trip ledger with win_rate / profit_factor / expectancy / MFE / MAE / holding-days lets the Go-Live Gate (4.5.4) evaluate exit quality and consistency -- metrics that Sharpe alone cannot surface.

## Success criteria (from masterplan 4.5.2)
1. round_trip_pairing_buy_to_sell_correct -- FIFO per-ticker pairing; partial exits split inventory proportionally; orphan SELLs dropped.
2. win_rate_and_profit_factor_match_quantified_strategies_definitions -- industry-standard formulas (Tharp 2007, QuantifiedStrategies.com).
3. mfe_mae_tracked_per_position_snapshot -- `mark_to_market` updates position `mfe_pct` monotonically up and `mae_pct` monotonically down.
4. performance_endpoint_includes_all_fields -- `/performance` inlines `round_trip_summary` dict with the new metrics.
5. evaluator_critique_pass.

## Design

**Migration (`scripts/migrations/add_round_trip_schema.py`)**: idempotent. Adds nullable columns:
- `paper_positions` +mfe_pct +mae_pct
- `paper_trades` +round_trip_id +holding_days +realized_pnl_pct +mfe_pct +mae_pct +capture_ratio
- New table `paper_round_trips` (canonical exit-quality ledger).

**Code paths:**
- `paper_trader.mark_to_market`: updates `mfe_pct = max(prev, pnl_pct)`, `mae_pct = min(prev, pnl_pct)` on each cycle.
- `paper_trader.execute_sell`: enriches the SELL trade dict with holding_days / realized_pnl_pct / mfe_pct / mae_pct / capture_ratio; writes canonical row to `paper_round_trips`.
- `_safe_save_trade` / `_safe_save_position`: detect schema errors and retry without new fields so pre-migration environments keep writing old rows (no hard-fail on deploy).
- `paper_round_trips.pair_round_trips`: reads historic trades, FIFO-pairs per ticker.
- `paper_round_trips.summarize`: win_rate, profit_factor = gross_win$ / |gross_loss$|, expectancy = WR*avg_win + (1-WR)*avg_loss, median_holding_days.

**Endpoints:**
- `GET /api/paper-trading/round-trips` (new) -- full ledger + last-50 table.
- `GET /api/paper-trading/performance` (modified) -- adds `round_trip_summary` field; existing fields unchanged.

## Anti-patterns guarded
- LIFO pairing biases holding-period stats toward short trades -- we use FIFO (industry standard; Tharp 2007).
- Profit factor undefined when gross_loss == 0 -- return 0.0 (not +inf) to keep JSON-serializable.
- MFE/MAE reset on new lots: tracked only on the currently-open position row; a new entry after full exit starts with prev = 0.

## Out of scope
- Intra-bar MFE/MAE (we sample at cycle cadence, not tick).
- Corporate actions (split-adjusted pairing) -- phase-level out-of-scope item.

## Risk
- ALTER TABLE on live BQ tables: run migration in low-traffic window. Idempotent, so reruns are safe.
- Large trade histories: `pair_round_trips` is O(n) in chronological scan; 2000-trade cap on endpoint.
