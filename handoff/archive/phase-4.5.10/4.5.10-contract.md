# Step 4.5.10 Contract -- Tests + reality-gap harness integration

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`
**Research tier:** simple

## Hypothesis
A focused test suite (unit tests on `compute_reconciliation()` + smoke tests per v2 endpoint) plus a separate "Reconciliation" line in every `handoff/harness_log.md` cycle entry gives us the regression protection and reality-gap visibility needed to close Phase 4.5. Keeping the reconciliation line separate from the scored Reality-Gap dimension preserves both signals (friction-model quality vs live-execution fidelity) without conflating them.

## Success criteria (from masterplan 4.5.10)
1. unit_tests_cover_metrics_v2_endpoints -- pytest file exercises metrics-v2 + round-trips + gate + reconciliation + live-prices + kill-switch + cycle-health + mfe-mae-scatter endpoints.
2. integration_test_covers_full_cycle_dry_run -- a single test invokes `python scripts/harness/run_harness.py --dry-run --cycles 1` equivalent programmatically; asserts the log gets a new cycle entry with a Reconciliation line.
3. reality_gap_metric_logged_to_handoff_harness_log -- `run_harness.py` appends `- Reconciliation: divergence=X.XX% alert=False (threshold=5.0%)` to every cycle entry (separate line; does not replace the scored Reality Gap).
4. evaluator_critique_pass.
5. no_regressions -- existing tests and endpoint shapes unchanged.

## Research findings (full entry in RESEARCH.md 4.5.10)

- Log format: add `- Reconciliation: divergence=X.XX% alert=False|True (threshold=5.0%)` after the scored block; keep the existing `- Reality Gap: N/10` line. Do NOT collapse the two (MLflow / MadeWithML consensus: divergence is a separate tracked metric).
- Test sizing pyramid (FastAPI 2025-2026): unit per service + success/failure per endpoint + one integration test. No percentage-coverage target.
- Mock pattern: `app.dependency_overrides` is FastAPI-canonical; fallback = import-site `patch()` when BQ is a module-level singleton (as in this codebase).
- Gate behavior: `alert=True` -> log `[WARN]` + stderr; **do NOT auto-revert cycle verdict** (anti-pattern -- divergence can be fill-price noise, not a strategy bug).
- Anti-pattern: testing only HTTP status codes on summary endpoints -- hides transform-layer regressions. Mitigation: unit-test `compute_reconciliation()` against fixtures that assert series values + `alert` bool, not just status.

## Design

**`backend/tests/test_paper_trading_v2.py` (new):**
- `FakeBQ` helper reused from earlier smoke tests.
- `TestReconciliationUnit`: 3 cases on `compute_reconciliation` -- no data, below-threshold divergence, above-threshold (alert).
- `TestV2Endpoints`: one test per v2 endpoint hitting GET (or POST with confirmation) and asserting response shape + key subset.
- `TestRealityGapLogging`: monkeypatches `compute_reconciliation` and exercises the harness log append path; asserts the `- Reconciliation:` line is present in the file.
- `TestNoRegression`: asserts existing endpoints (`/status`, `/portfolio`, `/trades`, `/snapshots`, `/performance`, `/run-now`) still return 200 with the pre-Phase-4.5 field shape.
- Uses `patch('backend.api.paper_trading.BigQueryClient', FakeBQ)` + `patch('backend.services.*.SomeFn', ...)` where needed (import-site pattern -- the BQ client is a module-level constructor in this codebase). Flagged as tech debt in the README.

**`scripts/harness/run_harness.py` (modify):**
- At cycle end (after the 4-score block writes), compute or read reconciliation summary and append one line. Best-effort: swallow errors so a reconciliation failure does not block the cycle log.

## Anti-patterns guarded
- Reconciliation line separate from the scored block -- different signals stay separate.
- `[WARN]` prefix on alert=True but no verdict mutation -- per research gate guidance.
- Unit tests on `compute_reconciliation()` itself (not just the endpoint) -- mitigates the "status code only" anti-pattern.

## Out of scope
- DI refactor of BigQueryClient into `Depends()` -- flagged for a future phase.
- Load testing / p95 latency (separate phase-5 SLO track).
- `httpx.AsyncClient` + `pytest-asyncio` -- TestClient suffices for these endpoints.

## Risk
- `run_harness.py` changes must remain idempotent and non-blocking. The reconciliation append path must catch any exception and no-op on empty data.

## Verification command
`source .venv/bin/activate && python -m pytest backend/tests/test_paper_trading_v2.py -q`
