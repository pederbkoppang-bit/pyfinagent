# Step 4.5.7 Contract -- Kill-switch v2 (Pause / Resume / Flatten-all + limits)

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`
**Research tier:** moderate (new subtopic: practitioner thresholds + UX patterns)

## Hypothesis
Bringing prop-firm-style hard limits (4% daily loss, 10% trailing DD) with auto-flatten-on-breach plus manual Pause/Resume/Flatten-all and an immutable JSONL audit trail gives operators a production-grade kill-switch. Manual actions use single-modal confirmation (industry standard for internal systems); auto-triggered actions require zero confirmation.

## Success criteria (from masterplan 4.5.7)
1. flatten_all_endpoint_closes_every_open_position -- `PaperTrader.flatten_all()` iterates all positions and issues a SELL at last-known price.
2. daily_loss_limit_configurable_default_4pct -- `settings.paper_daily_loss_limit_pct = 4.0`, wired through `evaluate_breach()`.
3. trailing_drawdown_limit_configurable_default_10pct -- `settings.paper_trailing_dd_limit_pct = 10.0`, EOD basis from rolling high-water mark.
4. breach_triggers_auto_flatten_and_pause -- `trader.check_and_enforce_kill_switch()` is called at the top of every cycle; on breach, auto-flatten + auto-pause.
5. all_actions_require_confirmation_modal_in_ui -- POST endpoints require `confirmation == "PAUSE"/"RESUME"/"FLATTEN_ALL"`; frontend `KillSwitchPanel` renders a single confirm modal.
6. evaluator_critique_pass.

## Research findings (full entry appended to RESEARCH.md)

- Daily loss limit 4% is the modal prop-firm default (FTMO 5%, FXIFY 4%, Alpha Capital 4%, FundedNext 4%; TradersSecondBrain Cheatsheet 2026 -- 3/4/5% cluster). For long-only unlevered equity, 4% sits at the conservative end of the moderate band and is consistent with 2x the canonical 2% per-trade rule (Van Tharp / CME).
- Trailing drawdown 10% EOD (end-of-day settled equity) is the upper-end standard for long-only equity signal systems. Intraday trailing would create noise for a daily-cycle strategy (Maven, Audacity, PropVator).
- Pause vs Flatten semantics: Pause = halt new entries, keep positions; Flatten = close all + cancel orders (Tradetron, exchange circuit-breaker literature). Resume precondition: both limits healthy.
- Manual kill-switch UX: single modal for internal systems (Tradovate/IBKR pattern); two-step (type-to-confirm) is consumer UX. Auto-triggered: zero confirmation.
- Anti-patterns: (a) soft limits that can be overridden (FINRA Rule 15c3-5 explicitly requires hard blocks); (b) missing audit log (3forge + ESMA Supervisory Briefing 2026 require timestamped append-only record).

## Design

**Settings (`backend/config/settings.py`):**
- `paper_daily_loss_limit_pct: float = 4.0`
- `paper_trailing_dd_limit_pct: float = 10.0`

**State + audit (`backend/services/kill_switch.py`):**
- `KillSwitchState`: thread-safe, replays audit log on startup to recover `paused` / `sod_nav` / `peak_nav`.
- `evaluate_breach(current_nav, daily_loss_limit_pct, trailing_dd_limit_pct)` -> dict with booleans + percentages.
- Audit JSONL at `handoff/kill_switch_audit.jsonl`, append-only, one event per line.

**PaperTrader hooks (`backend/services/paper_trader.py`):**
- `flatten_all(reason="manual_flatten")` -- iterates positions, issues SELLs at last-known price; returns `{closed_count, closed, reason}`.
- `check_and_enforce_kill_switch()` -- ratchet peak, snapshot SOD if unset, call evaluate_breach; on breach auto-flatten + pause.

**Autonomous loop (`backend/services/autonomous_loop.py`):**
- Step 5.5 (after mark_to_market): `check_and_enforce_kill_switch()`. If breached or already paused, short-circuit -- no Decide/Execute, still snapshot, return early.

**Endpoints (`backend/api/paper_trading.py`):**
- `GET /kill-switch` -- read-only state + breach + thresholds.
- `POST /pause` with body `{confirmation: "PAUSE"}`.
- `POST /resume` with body `{confirmation: "RESUME"}` -- 409 if a limit is still breached.
- `POST /flatten-all` with body `{confirmation: "FLATTEN_ALL"}` -- flattens then auto-pauses (known-quiet state per NYIF / 3forge).

**Frontend (`frontend/src/components/KillSwitchPanel.tsx` + paper-trading page):**
- Status banner with paused/active badge, breach percentages vs thresholds, 3 action buttons (visible based on state).
- `ConfirmModal` sub-component: single-modal confirmation for each action.
- Polls `/kill-switch` every 60s; 5-consecutive-failure threshold to surface an error.

## Anti-patterns guarded
- Hard-block endpoints (not soft) -- breach triggers auto-flatten unconditionally; can't be overridden from the caller side.
- Append-only audit log with timestamp + trigger + details.
- Resume precondition: both limits healthy; otherwise 409.
- Auto-triggered actions require zero confirmation; manual actions require single-modal.

## Out of scope
- Intraday trailing drawdown (strategy is cycle-based; EOD trailing is correct).
- BQ-backed audit table (JSONL on disk is sufficient for paper trading; upgrade is straightforward once schema migrations can run in prod).
- Two-step "type the word" confirmation (internal paper trading, not consumer platform).
- Multi-account flatten (we have a single `default` portfolio).

## Risk
- JSONL audit is local to the backend process; multi-replica deployments need a BQ-backed version later.
- `_last_run` / `_last_result` globals in the halted branch of autonomous_loop rely on the existing `global` declaration at line 64.
