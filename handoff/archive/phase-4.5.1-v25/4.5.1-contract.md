# Step 4.5.1 Contract — PSR / DSR / Sortino / Calmar + Bootstrap CI (metrics-v2)

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`

## Hypothesis
Adding peer-reviewed probabilistic Sharpe (PSR), deflated Sharpe (DSR), Sortino, Calmar, and a bootstrap 95% CI on rolling Sharpe to the paper-trading dashboard provides the statistical rigor required for the Go-Live Gate (step 4.5.4) without double-counting work already done in `perf_metrics.py`.

## Success Criteria (from masterplan 4.5.1)
1. PSR formula matches Bailey & Lopez de Prado (2012), Eq. 9.
2. DSR accounts for N strategies tested (Bailey & Lopez de Prado 2014, Eq. 8 threshold).
3. Bootstrap 95% CI on rolling Sharpe implemented (percentile method, N=1000 resamples; block-bootstrap fallback if |lag-1 autocorr| > 0.2).
4. `GET /api/paper-trading/metrics-v2` returns 200 with the expected response shape.
5. Evaluator critique pass (qa-evaluator subagent).

## Design

### Formulas (authoritative — cited in code docstrings)

**PSR** (Bailey & Lopez de Prado 2012, Eq. 9):
```
PSR(SR*) = Φ( (SR_hat - SR*) * sqrt(n - 1) / sqrt(1 - γ3*SR_hat + (γ4 - 1)/4 * SR_hat²) )
```
- γ4 is **raw** kurtosis (normal = 3). scipy's default returns excess kurtosis → add 3.
- Guard denominator: `max(1e-8, var_term)` to prevent zero-division at large SR.

**DSR threshold** (Bailey & Lopez de Prado 2014, Eq. 8):
```
SR* = sqrt(Var[SR]) * ((1 - γ_euler)·Φ⁻¹(1 - 1/N) + γ_euler·Φ⁻¹(1 - 1/(N·e)))
```
- γ_euler ≈ 0.5772156649
- Var[SR] = cross-sectional variance of Sharpe ratios across the N trials tested.
- Then DSR = PSR(SR*) with the threshold above.

**Sortino**: `mean(r) / std(r[r<0])` (MAR=0).
**Calmar**: `annualized_return / |max_drawdown|`.
**Bootstrap CI**: percentile method, N=1000 resamples, (2.5, 97.5). Fallback to stationary block bootstrap (block size ≈ √T) when |lag-1 autocorr| > 0.2.

### File layout

- **`backend/services/perf_metrics.py`** (extend):
  - `compute_psr(returns, sr_star=0.0) -> float`
  - `compute_dsr(returns, all_trial_sharpes, n_trials) -> float`
  - `compute_sortino(returns, mar=0.0) -> float`
  - `compute_calmar(returns, periods_per_year=252) -> float`
  - `compute_rolling_sharpe_bootstrap_ci(returns, window=60, n_resamples=1000, ci=0.95) -> tuple[float, float]`

- **`backend/services/paper_metrics_v2.py`** (new):
  - `compute_metrics_v2(bq) -> dict` — pull snapshots + trades from BQ, call perf_metrics, return dict.
  - `persist_metrics_v2(bq, metrics) -> None` — write row to `pyfinagent_pms.paper_metrics_v2` (best-effort; swallow BQ errors).
  - Thin orchestrator (no math inline — all math imported from perf_metrics per convention).

- **`backend/api/paper_trading.py`** (extend):
  - `@router.get("/metrics-v2")` → returns dict: `{psr, dsr, sortino, calmar, rolling_sharpe, ci_low, ci_high, n_obs, n_strategies_tested, computed_at}`.
  - Cache via `api_cache` with 60s TTL (consistent with `/performance`).

## Research Gate
- 3+ authoritative sources cited in code docstrings (Bailey 2012, Bailey 2014, Efron & Tibshirani 1993).
- Anti-pattern: excess-vs-raw kurtosis and denominator zero-division — both guarded.
- Hypothesis falsifiable: verification command parses file; endpoint returns 200 in a dry-run.

## Out of scope
- Annualization (raw per-period Sharpe is correct for PSR/DSR; existing annualized Sharpe remains in `/performance`).
- Persisting every 60s — metrics-v2 table row written at most once per cycle.
- Downside beta, Treynor, Omega — deferred unless Go-Live Gate needs them.

## Risk
- If n_obs < 30, return `note: "insufficient_data"` and skip PSR/DSR (both are unstable at small n).
- BQ table `paper_metrics_v2` must be created idempotently on first write (CREATE TABLE IF NOT EXISTS).
