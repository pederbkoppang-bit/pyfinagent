### THE crux — per-strategy PBO from one backtest: resolved
- **A single backtest's walk-forward WINDOWS CANNOT be the CSCV columns.** Bailey Algorithm 2.3 (verbatim): "each column n=1..N represents a vector of profits and losses associated with a particular model *configuration*"; the IS/OOS split is over ROWS (time). One PnL series → `compute_pbo` (analytics.py:205, `N<2` guard) returns **0.0**, which **PASSES** the `pbo<=0.20` gate → a false pass. WindowResult.total_return_pct is hardcoded 0; only `w.sharpe_ratio` is real, giving T=n_windows (~3-9) << the S*2=32 floor → another silent 0.0. Window-scalars are doubly unusable.
- **Chosen (textbook-exact, drop-in): per-strategy K-variant param grid.** For each rotation strategy, run K~8 param variants around its seed (same `strategy` categorical, jittered risk knobs), derive `daily_returns = np.diff(navs)/navs[:-1]` from each `result.nav_history` (~500 rows), stack as the N=K COLUMNS of a (T×K) matrix → existing `compute_pbo(matrix, S=16)`. Columns = competing configs of THAT strategy = exactly Bailey's CSCV definition. The K variants also supply DSR's `num_trials` + cross-sectional variance.
- **CPCV multi-path is the next-cycle upgrade** (de Prado AFML Ch.12; `cpcv_folds` already exists at `gate.py:42`; Arian/Norouzi/Seco 2024 = lowest PBO/highest DSR vs K-Fold) — but it yields a robustness OOS-Sharpe *distribution*, not the rank-degradation PBO scalar the gate consumes, so it complements (not replaces) the param-grid. Ship the grid now.
- **DSR num_trials** = number of INDEPENDENT (clustered) trials; plain count over-deflates (SAFE direction); effective-N (ONC) clustering deferred. Pass `num_trials = K` (grid size) / selector default 5. Threshold needs V[SR_n]>0 (per-window Sharpe variance; falls back to 0.5 if <2 windows → ensure ≥2 windows).

## Internal audit — engine/analytics (verified, exact)
- `generate_report(result, num_trials=1)` (`analytics.py:536`) → `["analytics"]` has `sharpe` (raw float), `deflated_sharpe` (DSR ∈[0,1]), `dsr_significant`, `total_return_pct`, `max_drawdown`, `hit_rate`, `n_trades`, `n_windows`, `num_trials`. **It does NOT compute PBO** — only DSR+Sharpe. So the gate's `dsr` ← `analytics["deflated_sharpe"]`; the gate's `pbo` ← a SEPARATE `compute_pbo` call.
- `compute_pbo(pnl_matrix, S=16) -> float` (`analytics.py:184`): input (T,N), T rows = time, N cols = configs; **silently returns 0.0 if N<2 OR T<S*2** (default needs T≥32, N≥2); S must be even. ← the load-bearing guard target.
- `compute_deflated_sharpe(observed_sr, num_trials, variance_of_srs=0.5, ...)` (`analytics.py:239`); generate_report wraps it (observed_sr=aggregate_sharpe, variance from per-window Sharpes).
- `BacktestResult` (`backtest_engine.py:110`): `windows: list[WindowResult]`, `aggregate_sharpe`, `nav_history: [{date,nav,cash}]` (~500 rows), `strategy_params`, `feature_importance_mdi/mda`, `all_trades`. `WindowResult` (`:89`): `window_id, train/test_start/end, sharpe_ratio` (real OOS), `total_return_pct`=0 (hardcoded), `max_drawdown_pct`, `hit_rate`, `num_trades`, `n_candidates/n_train_samples/n_features`, `feature_importance_mdi/mda`, `predictions`.
- `STRATEGY_REGISTRY` (`backtest_engine.py:32`): {triple_barrier, quality_momentum, mean_reversion, factor_model, meta_label}. Constructor validates `strategy` (silent fallback to triple_barrier on unknown) — the adapter must validate FIRST and reject (skip), never silent-fallback.
- Param application precedents: `run_harness.make_engine(params,...)` (constructor kwargs, fresh engine per trial — but threads only a SUBSET: NO target_vol/trailing/blend) vs `quant_optimizer._apply_params_to_engine` (in-place setter via `engine._strategy_params`). **Risk:** the vanilla make_engine factory silently ignores tb_risk_managed's target_annual_vol/trailing_* → it'd collapse onto tb_baseline. The adapter is factory-agnostic (caller supplies the factory); factory-extension is flagged.
- Warm-cache: one engine, `run_backtest(skip_cache_clear=True)` per run, ONE `cache.clear_cache()` at end (`quant_optimizer.run_loop` precedent). **Macro preload IS inside run_backtest** (`backtest_engine.py:299-302`) → adapter needs NO own preload; the ~40min hang only bites paths that bypass run_backtest. `run_backtest` calls `trader.full_reset()` per run (independence).
- Mockability: build the REAL `BacktestResult` + 2-3 REAL `WindowResult` (with non-zero `.sharpe_ratio`) + ~40 synthetic nav rows; run the REAL `generate_report`+`compute_pbo` (pure numpy, $0). `compute_pbo` testable in isolation with a hand-built (T≥32, N≥2) matrix.

## Design (the cycle's plan)
- File: `backend/autoresearch/strategy_backtest_adapter.py`. Factory `make_engine_backtest_fn(engine_factory, *, num_param_variants=8, param_grid_fn=None, num_trials=None, pbo_S=16, min_pbo_rows=32, logger=None) -> backtest_fn(params) -> {dsr, pbo, sharpe, n_variants, n_windows}` — exactly the producer's injected `BacktestFn` boundary.
- Pure helpers: `_daily_returns_from_nav`, `_default_param_grid` (jitter risk knobs, keep strategy categorical, validate name), `_assemble_pbo_matrix` (truncate to shortest common length, drop empties, **return None if N<2 or T<min_rows** — the load-bearing guard), `_extract_dsr_sharpe` (wraps generate_report).
- Undersize guard: matrix None → return `{dsr, sharpe}` WITHOUT `pbo` → producer SKIPS (never emit pbo=0.0). Per-variant try/except so one failing variant degrades column count.
- **Completable this cycle ($0 mock-tested):** the adapter + helpers + a full mock test (REAL generate_report+compute_pbo on a hand-built fake BacktestResult; undersize-guard skip; strategy-name reject; end-to-end registry→adapter→producer→selector with a mocked engine_factory) + one `@pytest.mark.skip` opt-in live integration test.
- **DEFERRED:** the LIVE multi-run bake-off (4 seeds × K ≈ 32+ real backtests, tens of minutes — too long for an in-session $0 step); CPCV multi-path; the weekly cron; the params→settings.paper_* deployment bridge; effective-N clustering; a true date-keyed matrix join (length-truncation is a benign proxy since variants share the bdate grid).
