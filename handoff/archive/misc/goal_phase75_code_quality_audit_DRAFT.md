## Interpretation

- **Session mode:** ultracode (multi-agent Workflow orchestration), Fable 5 reasoning on
  the IT-stack auditor roles (they inherit the session model on the $0 Claude Code Max
  rail). Layer-3 Researcher + Q/A stay on their steady-state `model: opus` / `effort: max`
  pins per the 67.4 revert doctrine — the auditors are workflow fan-out agents, not
  Layer-3 harness roles, so the exactly-3-agents doctrine is unchanged.
- **Scope:** the ENTIRE codebase, one auditor per IT-stack role — backend/Python+FastAPI,
  frontend/Next.js+React+TS, data engineering/BigQuery, security, DevOps/ops-scripts,
  QA/testing, architecture, performance, LLM-app engineering, dependency/supply-chain,
  docs/config drift. Audited AGAINST official documentation + current best practices
  (research gate arms each role with citable doc anchors).
- **Deliverable:** verified findings register + NEW masterplan remediation steps
  (executor-tagged, pending) so cheaper sessions can execute — the phase-72/73 queue
  pattern.

  touches, no flag flips, no optimizer runs (`historical_macro` stays FROZEN), paper-only.
- $0 metered — Claude Code Max rail only.
- Do-no-harm: kill-switch limits / stops / sector caps / DSR>=0.95 / PBO<=0.5 stay

- No duplication of the existing pending queues (phase-72 remediation, phase-73 build

  dedupes against all of them.
- Every finding requires verbatim code evidence + a doc/best-practice basis; adversarial
  verification before it can spawn a step. Full 5-file protocol; harness stays exactly
  3 agents (Main + Researcher + Q/A).
