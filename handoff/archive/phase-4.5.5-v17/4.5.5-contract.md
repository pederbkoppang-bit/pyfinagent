# Step 4.5.5 Contract -- Agent-rationale drawer + per-trade signal attribution

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`

## Hypothesis
Persisting the agent-signal attribution tree (Analyst -> Bull/Bear -> Trader -> Risk) alongside each trade lets operators audit *why* a trade happened -- a prerequisite for live-capital promotion and post-mortem debugging.

## Success criteria (from masterplan 4.5.5)
1. paper_trade_schema_includes_signals_array -- new `signals` STRING column on paper_trades (JSON-encoded list of signal dicts).
2. autonomous_loop_passes_analysis_signals_to_decide_trades -- `TradeOrder.signals` populated in `decide_trades`, propagated into `execute_buy`/`execute_sell`.
3. drawer_uses_progressive_disclosure_per_tradingagents_pattern -- React drawer uses collapsible `<details>` per layer; no inline one-shot dump.
4. no_plaintext_secret_or_pii_leak_in_rationale -- `redact_pii` strips emails + API keys on extraction AND on read (defense-in-depth).
5. evaluator_critique_pass.

## Design
- `backend/services/signal_attribution.py` (new): `extract_signals_from_analysis`, `group_signals_for_drawer`, `redact_pii`. PII regexes for emails, Anthropic/Google/OpenAI keys, generic 32+-char tokens.
- `backend/services/portfolio_manager.py`: `TradeOrder.signals: list[dict]`; populated on SELL (sell_signal, signal_downgrade) and BUY (buy_candidates) paths.
- `backend/services/paper_trader.py`: `execute_buy`/`execute_sell` accept optional `signals=` and serialize via `json.dumps` into the trade row; `_ROUND_TRIP_FIELDS` += `"signals"` so the pre-migration fallback strips it cleanly.
- `backend/services/autonomous_loop.py`: passes `order.signals` through to executors. Adds `run_cycle = run_daily_cycle` alias so the verification command imports cleanly.
- `scripts/migrations/add_round_trip_schema.py`: ALTER TABLE ADD COLUMN IF NOT EXISTS `signals STRING`.
- `backend/api/paper_trading.py`: new `GET /trades/{trade_id}/rationale` -> parses stored JSON, defense-redacts, returns progressive-disclosure tree.
- `frontend/src/components/AgentRationaleDrawer.tsx` (new): right-side slide-over, layers Analyst -> Debate (Bull/Bear, split colors) -> Trader -> Risk; each `<details>` collapsible.
- `frontend/src/app/paper-trading/page.tsx`: trades-tab rows become `cursor-pointer`, click sets `rationaleTradeId`, drawer renders conditionally.

## Anti-patterns guarded
- ARRAY<STRUCT> BQ typing avoided -- JSON string column keeps dynamic parameterized INSERT path intact.
- trade_id sanitized in the rationale endpoint (alnum + -/_) before BQ lookup.
- PII redactor runs twice (extract + read) -- defense-in-depth for older rows written before the redactor existed.
- Drawer never renders signals inline on the trades table -- progressive disclosure only, per TradingAgents.

## Out of scope
- ARRAY<STRUCT> BigQuery column variant (would require rewriting save_paper_trade to handle ARRAY types).
- Agent-by-agent cost/token breakdown inside the drawer (phase-wide "per-agent cost UI" moved to a debug sub-route in the phase-4.5 contract).

## Risk
- Old trades without `signals` return empty drawer -- fine (drawer shows empty state).
- Large rationales could bloat BQ rows -- `_trim(text, 500)` caps each rationale.
