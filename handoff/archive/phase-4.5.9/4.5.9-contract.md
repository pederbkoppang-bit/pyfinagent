# Step 4.5.9 Contract -- MFE/MAE scatter + Edge-Ratio per trade

**Drafted:** 2026-04-16
**Parent:** `handoff/current/phase-4.5-contract.md`
**Research tier:** simple (data captured by 4.5.2; step is chart surface + edge_ratio)

## Hypothesis
A scatter chart with |MAE| on X, MFE on Y, point color = win/loss, and capture_ratio encoded as color saturation surfaces the "winner that reversed" exit-leakage cluster visible in the round-trip ledger. Edge-Ratio = MFE / |MAE| per trade, averaged, gives a single-number summary operators can trend.

## Success criteria (from masterplan 4.5.9)
1. scatter_uses_mfe_x_mae_y_per_closed_round_trip -- masterplan wording is "mfe_x_mae_y"; mapped literally as "MFE and MAE are the two axes" (AFML convention puts MAE on X, MFE on Y). Each point = one closed round-trip.
2. capture_ratio_computed_pnl_over_mfe -- `capture_ratio = realized_pnl_pct / mfe_pct`, already populated by paper_round_trips.py; re-derived client-side as a safety net.
3. chart_surfaces_exit_leakage_pattern -- leakage coloring: capture_ratio < 0.4 AND mfe_pct > P75 flagged amber/red.
4. evaluator_critique_pass.

## Research findings (full entry in RESEARCH.md)

- Axis convention: AFML Ch. 13 community code uses MAE on X (positive, adverse cost) and MFE on Y (positive, favorable excursion). This makes top-right = "high opportunity but also high adverse excursion" (volatile, exit-sensitive), top-left = ideal entries.
- Edge Ratio: `mfe_pct / |mae_pct|` per trade; distinct from Curtis Faith/Build Alpha's ATR-normalized time-series E-Ratio (which is portfolio-level). Use the per-trade form.
- Leakage diagnostic (TradeWink, TradesViz): high-MFE + low-capture_ratio cluster is the "winner that reversed" signature. Rule: `capture_ratio < 0.4 AND mfe_pct > P75(mfe_pct)`.
- Color convention (TradesViz, TradingDiary Pro, TraderSync, Tradervue 2024-2026): green = realized_pnl>0, red = realized_pnl<=0; point size = holding_period; color saturation = capture_ratio tertile.
- Anti-pattern: closed-trades-only survivorship + force-close-distortion; TradingDiary Pro's explicit note that MAE understates true risk when external events (margin, EOD) closed positions.

## Design

**Backend (`backend/api/paper_trading.py`):**
- New `GET /api/paper-trading/mfe-mae-scatter` endpoint.
- Reuses `pair_round_trips()` + `summarize()` from `backend/services/paper_round_trips.py` (4.5.2 dependency).
- Response shape:
  ```json
  {
    "points": [{
      "ticker": "AAPL", "entry_date": "...", "exit_date": "...",
      "mfe_pct": 12.0, "mae_pct": -3.5, "capture_ratio": 0.83,
      "realized_pnl_pct": 10.0, "holding_days": 10,
      "leakage_flag": false
    }],
    "summary": {
      "edge_ratio": 2.4,             // mean(mfe_pct / |mae_pct|) across trades
      "avg_capture_ratio": 0.67,
      "mfe_p75": 8.2,                // 75th percentile of mfe_pct across all trades
      "leakage_threshold_capture": 0.4,
      "n_points": 42,
      "n_leakers": 3
    },
    "computed_at": "..."
  }
  ```
- Leakage flag computed server-side so the UI agrees with evaluator logic.
- 120s cache key `paper:mfe_mae_scatter`.

**Frontend (`frontend/src/components/MfeMaeScatter.tsx`, new):**
- Recharts `ScatterChart` with two datasets: winners (green) + losers (rose).
- X domain: 0 .. max(|MAE|); Y domain: 0 .. max(MFE).
- Point radius = `clamp(holding_days, 4, 16)`.
- Leakage points highlighted (amber stroke + larger radius).
- Header pills: `edge_ratio`, `avg_capture_ratio`, `n_leakers`.
- Dashed 45-degree reference line (MFE = |MAE|).
- Added as a new "Exit quality" tab on the paper-trading page (so positions/trades/chart/reality-gap tabs stay unchanged).

## Anti-patterns guarded
- Do NOT use Build Alpha's ATR-normalized E-Ratio formula -- this chart is per-trade, not time-series.
- Do NOT invert axes (MFE on X / MAE on Y inverts the "danger zone" semantics).
- Closed-trades-only bias: response notes that all points are closed round-trips; open positions are excluded by construction.

## Out of scope
- ATR normalization / intraday bar granularity (daily-cycle strategy).
- Force-close flag (would require a new enum on round-trip records; deferred).
- Historical comparison view (before/after parameter changes).

## Risk
- P75 threshold is brittle with n<10 trades; endpoint returns `n_leakers=0` and `mfe_p75=null` below that floor.
