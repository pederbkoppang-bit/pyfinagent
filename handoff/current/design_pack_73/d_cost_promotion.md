## 2. Cost-per-bp reporting — per-decision diagnostic, no double-count (build step 73.4.1)

Spec (verbatim from the gate):
- Minimum per-decision log row (New Quant §7.10 + arXiv:2607.10286 App C taxonomy Ct=Ct_llm+Ct_trd+Ct_inf+Ct_sto): {decision_id, cycle_id, ts, excess_return_bps, token_cost_usd, slippage_usd, fees_usd, cost_per_bp, latency_ms, model_version, pricing_snapshot_date}. The only genuinely-new field is per-decision cost_per_bp; the rest reuse llm_call_log + the session gauge + trade_statistics.
- cost_per_bp = total_cost_usd / max(excess_return_bps, eps), where total_cost = token_cost + slippage + fees and excess_return_bps = (strategy_return - benchmark_return)*1e4 realized per trade/round-trip (compute_round_trips/compute_trade_statistics already exist in analytics.py). Report None/inf-flag when excess_bps<=0, mirroring the /efficiency NULL-on-zero pattern (sovereign_api.py:567-570).
- NO double-count: the net-DSR series (Component 1) already subtracts cost ONCE as a return drag; cost_per_bp reports the SAME cost as a diagnostic ratio and is NOT fed back into DSR or any single scalar objective. DSR stays the objective; cost_per_bp is the report.
- Per-cycle token-cost derivation (three equivalents, all avoiding the gauge-SUM trap): (A) MAX(session_cost_usd) GROUP BY cycle_id on llm_call_log (gauge terminal value); (B) recompute per-call from input_tok/output_tok/cache_*_tok x MODEL_PRICING (cost_tracker.py:165-183) and SUM GROUP BY cycle_id (keeps per-model/per-ticker attribution); (C) SUM(cost_usd_est) on api_call_log (a real per-call cost column). NEVER SUM(session_cost_usd) across llm_call_log rows.
- Sit the per-decision cost_per_bp below the existing firm-level ratio /api/sovereign/efficiency profit_per_llm_dollar = realized_pnl_usd/(anthropic+vertex+openai) (sovereign_api.py:564-568) -- same idea at finer granularity; input!=output token rates (3-8x) are already priced separately in cost_tracker MODEL_PRICING (:20-80).

Integration seams: backend/services/observability/api_call_log.py: log_llm_call (:220-290) writes llm_call_log {token cols + session_cost_usd GAUGE + cycle_id, NO per-call cost col}; the api_call_log table (:14-52) has per-call cost_usd_est FLOAT64. | backend/agents/cost_tracker.py: AgentCostEntry.cost_usd (:191-204) = per-call cost primitive; MODEL_PRICING (:20-80) prices input!=output. | backend/backtest/analytics.py: compute_round_trips / compute_trade_statistics = realized excess_return_bps source. | backend/api/sovereign_api.py: /efficiency (:505-570) = existing profit_per_llm_dollar aggregation to sit above cost_per_bp. | backend/services/paper_trader.py: transaction_cost (:289/:472) = fees_usd source; slippage_usd = fill_price-close in alpaca_paper (:277), 0 in bq_sim.

Cost: Reporting-only, $0: reuses the session_cost gauge + llm_call_log + trade_statistics that already exist; the single new artifact is a per-decision cost_per_bp row/line, no new LLM calls. Watch-item to meter (not a cost of this component): the #2 reflection fan-out fires 4 LLM calls per closed trade on the flat-fee Gemini rail -- this reporting layer is exactly what should meter that token throughput for the #2/#3 pilots.
