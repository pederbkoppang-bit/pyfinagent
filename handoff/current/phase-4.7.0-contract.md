# Contract -- Cycle 68 / phase-4.7 step 4.7.0

Step: 4.7.0 Route inventory + 30-day usage telemetry

## Hypothesis

No pageview infrastructure exists today (confirmed by Explore: no
Plausible/PostHog/GA; middleware.ts is auth-only; perf_tracker is
backend-only). A git-activity 30-day proxy over `frontend/src/app/
**/page.tsx` discriminates sufficient signal (47 commits on /backtest
vs 1 on /login in the last 30 days) to unblock step 4.7.1's
"remove or merge zero-open pages" decision. We NAME the usage source
honestly -- `git_activity_30d` -- rather than inventing a page-view
number we cannot actually verify.

A pageview-tracker wiring (Option A from research) is a follow-up in
4.7.0's sibling step so future 30-day windows have real opens.

## Scope

Files created:

1. **NEW** `scripts/harness/frontend_route_inventory.py`
   - Walks `frontend/src/app/**/page.tsx` for route paths
   - Runs `git log --since=30d --name-only -- frontend/src/app` to
     build per-file commit counts (30d window)
   - Maps file -> route path (e.g., `app/backtest/page.tsx` -> `/backtest`)
   - Emits `handoff/frontend_usage.json` with {routes[], ...}

2. **NEW** `handoff/frontend_usage.json` (generated by #1)

## Immutable success criteria

1. `every_route_has_usage_count`: every route in
   `handoff/frontend_usage.json` has an integer `opens_30d` field
   (ZERO is a legitimate value -- a page with no edits in 30 days
   gets opens_30d=0, not missing field)
2. `usage_source_named`: top-level `usage_source` and each per-route
   `usage_source` populated with a non-empty string; the source
   identifier must be inspectable (e.g., `"git_activity_30d"`).

## Verification (immutable, from masterplan.json)

    test -f handoff/frontend_usage.json && \
    python -c "import json; d=json.load(open('handoff/frontend_usage.json')); \
      assert all('opens_30d' in r for r in d['routes'])"

## Discriminating content

- every route must appear (no filtering): /, /login, /agents,
  /analyze, /backtest, /compare, /paper-trading, /performance,
  /portfolio, /reports, /settings, /signals -- 12 routes (excl.
  api/auth).
- opens_30d MUST come from a computable source; write the command
  into the JSON so auditors can re-run it.
