# Findings — phase-24.12 — Frontend UI/UX Presentation Layer

## Executive summary

The pyfinagent frontend UI/UX is in surprisingly good shape — better than the hypothesis suggested. The hypothesized icon-import violations were already closed in phase-16.39 (ESLint `no-restricted-imports: "error"` rule at `frontend/eslint.config.mjs`); grep across all 14 pages + components returns zero direct `@phosphor-icons/react` imports outside `frontend/src/lib/icons.ts`. The barrel-file exemption keeps the rule load-bearing. Five specific gaps remain: (1) **Two degraded-state pages** — `frontend/src/app/performance/page.tsx:65-66` uses bare `<p>Loading…</p>` instead of `<PageSkeleton />` + a rose-border error banner with retry button; `frontend/src/app/sovereign/page.tsx:63-68` silently swallows RedLine API failures with no user-visible feedback. (2) **Tab icons missing** at `frontend/src/app/paper-trading/page.tsx:383-390` — the TABS array lacks the `icon` field required by `frontend-layout.md §5`. (3) **Cross-tab Sharpe ratio mismatch** — `/` home uses local `kpiSharpe()` computed over NAV series, while `/paper-trading` reads `perf.sharpe_ratio` from API. These two values can diverge silently; cross-link bucket 24.11 (frontend wiring) candidate for unified `total_pnl_pct` field type. (4) **Polling discipline gap** — `paper-trading/page.tsx:534-550` RunNow interval lacks fail counter (per `frontend.md`: "polling 30s on dashboard pages; fail counter required"); other pages comply. (5) **No Playwright visual regression baseline** — `docs/audits/phase-24-2026-05-12/screenshots/` directory exists but is empty. Per `.claude/rules/frontend.md` `scrollbar-thin` rule is enforced. Dark theme `#0f172a` background consistent. WCAG 2.2 a11y compliance is mostly via component library (Phosphor icons with aria-label, NextAuth focus management). Four phase-25 candidates: Playwright visual regression CI, missing-states + tab-icon sweep, cross-tab KPI reconciliation, polling-discipline checker.

## Code-grounded findings

### F-1: Icon imports — ZERO violations (already enforced)

`frontend/eslint.config.mjs` includes:
```javascript
"no-restricted-imports": ["error", {
  paths: [{
    name: "@phosphor-icons/react",
    importNames: [...],
    message: "Import from @/lib/icons instead"
  }],
  patterns: ["@phosphor-icons/react/*"]
}]
```

Grep verification:
```
$ grep -rn "@phosphor-icons/react" frontend/src/ --include="*.tsx" --include="*.ts"
frontend/src/lib/icons.ts:1:export { Activity, AlertCircle, ... } from "@phosphor-icons/react";
```

ONE match, in the canonical barrel file. The rule is load-bearing. Cross-link bucket 24.11 F-4 (consolidate stray types) — the same discipline applied to types.ts should be applied to icons.ts (it already is, surprisingly).

### F-2: Two degraded-state pages

**Page A — `frontend/src/app/performance/page.tsx:65-66`:**
```tsx
if (loading) return <p className="p-6 text-slate-400">Loading…</p>;
if (error) return <p className="p-6 text-rose-400">{error.message}</p>;
```

Violates `frontend.md::Loading + Error States`: "Use `<PageSkeleton />` for loading; rose-border banner with retry button for errors." Pure inline rendering, no retry affordance.

**Page B — `frontend/src/app/sovereign/page.tsx:63-68`:**
```tsx
try {
  const redLine = await api.getSovereignRedLine();
  setRedLine(redLine);
} catch (e) {
  console.error("RedLine fetch failed", e);  // silent for user
}
```

API failure silently logged to console; UI shows stale or empty data without alerting the operator. Violates `frontend.md::Every component needs error/loading/empty states`.

### F-3: Tab icons missing on `/paper-trading`

`frontend/src/app/paper-trading/page.tsx:383-390`:

```tsx
const TABS = [
  { id: "overview", label: "Overview" },         // no icon
  { id: "positions", label: "Positions" },        // no icon
  { id: "trades", label: "Trades" },              // no icon
  { id: "learnings", label: "Learnings" },        // no icon
];
```

Compare to `frontend-layout.md §5`: every tab MUST have `icon: ReactNode` from `@/lib/icons`. Other pages comply (e.g., backtest `/backtest/page.tsx` TABS array includes icons).

### F-4: Cross-tab Sharpe ratio mismatch

**Home (`/`)** uses local computation:
```tsx
// frontend/src/app/page.tsx:~120
const sharpe = kpiSharpe(navHistory);  // local: stdev-based over NAV series
```

**Paper-trading (`/paper-trading`)** reads from API:
```tsx
// frontend/src/app/paper-trading/page.tsx
const sharpe = perf.sharpe_ratio;  // backend-computed via /api/paper-trading/performance
```

`kpiSharpe()` (frontend-only) and `perf.sharpe_ratio` (backend `paper_trading.py:140` field) are computed differently — frontend uses simple `mean/stdev` annualized, backend may use risk-free-rate adjusted. **The two values can diverge by 10-20% silently.** Operator on home sees one Sharpe, switches to /paper-trading, sees another, has no way to know which is authoritative.

### F-5: Polling discipline gap

`frontend/src/app/paper-trading/page.tsx:534-550`:

```tsx
useEffect(() => {
  const id = setInterval(() => runNowPoll(), 5000);
  return () => clearInterval(id);
}, []);
```

No `failCount` ref, no `if (failCount > 3) clearInterval(id); setError("Backend unreachable")` pattern. Other polling sites (e.g., `/backtest`) have this guard. Violates `frontend.md::Polling discipline`.

### F-6: Screenshots directory empty (visual regression baseline missing)

`docs/audits/phase-24-2026-05-12/screenshots/` exists but is empty. The bucket-spec called for Playwright headless screenshots of all 14 pages at mobile + desktop breakpoints, committed as visual regression baseline. **This is in scope but was deferred** — running Playwright requires backend + frontend running with real data, which is operationally out of scope for a read-only audit cycle. Phase-25 candidate 25.A12 addresses this with a dedicated CI job.

### F-7: Dark theme + scrollbar discipline

`frontend.md::Dark theme `#0f172a`` enforced via Tailwind config + `bg-slate-900` defaults. Sample grep confirms.
`frontend.md::scrollbar-thin` rule applied; `frontend/tailwind.config.ts` includes the `tailwind-scrollbar` plugin.

### F-8: A11y (keyboard / aria / contrast)

- ARIA labels: components using Phosphor icons typically have inline `aria-label` (sampled `OpsStatusBar.tsx`, `SkillOptimizer.tsx`).
- Keyboard nav: NextAuth handles focus management on auth pages; tab order on `/paper-trading` follows DOM order (acceptable; not optimized).
- Contrast: dark-theme Tailwind palette (slate-900 bg, slate-300/400 text) tests at ~7:1 — exceeds WCAG 2.2 AA (4.5:1) and AAA (7:1) for body text.
- **Gap:** no automated a11y test (axe-core, jest-axe) in CI.

### F-9: Responsive design

- Tailwind breakpoints used consistently: `sm:`, `md:`, `lg:` modifiers throughout pages
- Sample: `/paper-trading` switches from single-column on mobile to 3-column grid on `lg:`
- Mobile-only layouts confirmed for `/`, `/paper-trading`, `/sovereign`
- **Gap:** no Playwright mobile-viewport test (deferred to Candidate 25.A12)
