# Findings — phase-24.13 — Profit-Maximization Red-Line Alignment Synthesis

## Executive summary

Current pyfinagent behavior is mis-aligned with the red-line goal ("maximize profit at lowest cost; dynamically shift strategy to whichever is making the most money" — `project_system_goal.md`) across all four sub-goals. Synthesizing across **buckets 24.1-24.9** (all complete) reveals four compounding misalignments: (a) **anti-profit** — stop-loss orphan (bucket 24.1 F-1, `paper_trader.py:414`) holds TER at -12.30% (~-$1,107 accruing) with 6 of 11 positions (55% of portfolio) stop-less; `portfolio_manager.py:82-88` silently bypasses None stops via Python truthiness. (b) **anti-cost** — `llm_client.py` ignores `cost_budget.tripped` flag (bucket 24.8 F-4), `cost_tracker.py:147` `CACHE_WRITE_PREMIUM = 1.25` under-reports by 60% vs actual 2.0x (bucket 24.9 F-1), system prompt below 4096-token cache threshold so caching silently no-ops (bucket 24.9 F-2), and full-pipeline runs without persistence cost ~$1.50/day with zero usable output (bucket 24.2). (c) **anti-switching** — zero `autoresearch|meta_evolution` imports in `autonomous_loop.py` (bucket 24.3 F-1, grep-verified); `monthly_champion_challenger.py:76` hard-codes `actual_replacement: False`; `autoresearch/cron.py:29-38` registers `func=lambda: None`; the goal-c strategy-switching mechanism does not exist. (d) **unobservable** — `sovereign_api.py:394-395` hardcodes `anthropic: 0.0, vertex: 0.0, openai: 0.0` LLM cost vs P&L; the primary leaderboard BQ view (`pyfinagent_pms.strategy_deployments`) does not exist (`sovereign_api.py:336` self-disclosure); no `profit_per_llm_dollar` metric exists anywhere in the codebase. Industry context (arxiv 2503.21422, March 2025 survey): NO published trading system has this metric — pyfinagent would be a first mover. Dollar estimates: stop-loss orphan ~$1,107 unrealized (TER alone); full-pipeline waste ~$1.50/day; LLM cost under-count ~$13.50/month; 62 consecutive plateau experiments since 2026-04-21 = wasted backtest compute. Three new phase-25 candidates close goal-c + goal-d gaps: 25.Q profit-per-LLM-dollar metric, 25.R strategy auto-switching policy, 25.S daily P&L attribution report (per SHARP arxiv 2605.06822 attribution is load-bearing for alpha).

### F-4: Cost budget enforcement path — broken

`backend/api/cost_budget_api.py` correctly:
- Tracks per-provider spend
- Sets `tripped=True` when threshold breached
- Stores state in BQ `cost_budget_state` table

`backend/services/llm_client.py` does NOT:
- Check `cost_budget.tripped` before any API call
- Raise `BudgetBreachError` to abort cycle

Result: after a budget breach at 11pm, the next day's 7am cycle runs LLM calls normally, accumulating MORE spend on top of the breach. **The budget is observational, not enforcement.** Cross-link bucket 24.8 candidate 25.A8 (cost-budget hard-block).
