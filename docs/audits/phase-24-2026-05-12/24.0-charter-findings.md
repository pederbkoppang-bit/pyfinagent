## External-research summary

### Source 1: Anthropic — Harness Design for Long-Running Application Development
**URL [accessed 2026-05-12]:** https://www.anthropic.com/engineering/harness-design-long-running-apps

Establishes the file-based-handoff pattern with context resets, the three-phase Plan→Generate→Evaluate cycle, and the stress-test doctrine: "every component in a harness encodes an assumption about what the model can't do on its own, and those assumptions are worth stress testing." The key warning that drives bucket 24.4 (aliasing): "agents tend to respond by confidently praising the work — even when, to a human observer, the quality is obviously mediocre." Phase-24 follows this paper's pattern directly: handoff files (`handoff/current/`) + context-resetting subagent spawns + bucket-by-bucket audit cycles.

### Source 2: Anthropic — Building Effective AI Agents
**URL [accessed 2026-05-12]:** https://www.anthropic.com/engineering/building-effective-agents

Establishes "find the simplest solution possible, and only increasing complexity when needed" — the philosophical basis for bucket 24.2's test of "does the 28-agent pipeline earn its compute cost vs lite mode?" Also documents the orchestrator-workers pattern and evaluator-optimizer pattern; the phase-24 audit applies evaluator-optimizer at the bucket level (Q/A judges Main's findings doc).

### Source 3: Anthropic — Multi-Agent Research System
**URL [accessed 2026-05-12]:** https://www.anthropic.com/engineering/built-multi-agent-research-system

Describes the LeadResearcher (Opus 4) + Sonnet subagent topology that scored 90.2% over single-agent on internal benchmark. Key quote: "subagents write structured results to external storage" to avoid "game of telephone." Phase-24's five-file handoff protocol directly mirrors this: research_brief.md → contract.md → findings doc → experiment_results.md → evaluator_critique.md, each fresh-spawned, each writing to disk for the next phase to read.

### Source 4: Claude Code — Hooks Reference
**URL [accessed 2026-05-12]:** https://code.claude.com/docs/en/hooks

Catalogs all hook categories: PreToolUse, PostToolUse, PostToolUseFailure, InstructionsLoaded, Stop, TaskCompleted, SubagentStart/Stop, SessionStart/End, ConfigChange. The phase-24 audit relies on these hooks for protocol enforcement: `archive-handoff` (PostToolUse on masterplan flip) moves files to `handoff/archive/phase-X.Y/`; `auto-commit-and-push.sh` (PostToolUse) auto-commits and pushes per step; `live_check_gate.py` gates the push on operator-confirmed live evidence. Bucket 24.10 audits MCP deny rules which are hook-enforced.

### Source 5: arXiv 2603.13942v1 — AI Agents in Financial Markets: Architecture, Applications, and Systemic Implications
**URL [accessed 2026-05-12]:** https://arxiv.org/html/2603.13942v1

Introduces the Agentic Financial Market Model (AFMM) — a four-layer framework (Data Perception, Reasoning Engine, Strategy Generation, Execution+Control) that maps onto pyfinagent's 4-layer architecture in `ARCHITECTURE.md`. Formalizes five control dimensions including "Supervisory Observability." Key quote: "agentic systems should incorporate built-in control mechanisms such as approval gates, exposure limits, and emergency shutdown procedures." Directly informs buckets 24.1 (stop-loss approval gate is orphan), 24.8 (kill-switch + cost-budget), and 24.5 (Slack alerting as supervisory observability).

### Source 6: arXiv 2512.02227v1 — Orchestration Framework for Financial Agents
**URL [accessed 2026-05-12]:** https://arxiv.org/html/2512.02227v1

Provides a blueprint matching pyfinagent's layer-1 + layer-2 topology: planner, orchestrator, alpha, risk, portfolio, backtest, execution, audit, memory agents. Mandates strict LLM/data-leakage separation: "LLM agents never receive test-period returns, prices, or labels." Walk-forward requires "at least a 2-minute gap between feature timestamps and labels." Drives bucket 24.6 audit of `backend/backtest/backtest_engine.py` walk-forward correctness.
