# Findings — phase-24.6 — Backtest Engine + Walk-Forward + Live-vs-Backtest Reconciliation

## Executive summary

The backtest engine is structurally sound — `backend/backtest/backtest_engine.py` (~900 LOC) implements walk-forward with embargo and uses gradient boosting + 5 strategies + MDA (Mean Decrease Accuracy) feature importance caching. The harness drives sub-period evaluation (2018-2020, 2020-2022, 2023-2025) at `scripts/harness/run_harness.py` and gates on DSR ≥ 0.95 + PBO ≤ 0.20 + reality-gap ≥ 0.5 per Lopez de Prado AFML Ch. 12. Seed stability is hardcoded `random_state=42` at four sites (`backtest_engine.py:725,749,886,914`) ensuring deterministic re-runs; a `GET /api/backtest/harness/seed-stability` endpoint exists at `backtest.py:1330` and reads from `handoff/data/seed_stability_results.json` — but this file is NOT in git status, suggesting the seed-stability test has not been run recently or its output is stale. The most material gap is the live-vs-backtest reconciliation: `paper_go_live_gate.py:91-94` honestly comments "Use divergence as a proxy for Sharpe gap when explicit backtest Sharpe isn't available — stays conservative." NO code path computes `live_realized_sharpe - backtest_predicted_sharpe` explicitly; the SR-gap check is satisfied by NAV divergence, which is correlated but not equivalent. MDA feature importances flow backtest → live via `mda_cache.json` (one-directional channel — confirmed working). NO channel flows live paper-trading outcomes back into optimizer warmstart or strategy selection. The optimizer is in a 62-experiment plateau (run `0083971f`, all dated 2026-04-21, all discarded, baseline Sharpe 1.1705 / DSR 0.9526) — the planner's Rule 1 (plateau after 10 consecutive discards) should have triggered strategy-switch at exp 11 but the run continued to 62, suggesting standalone optimizer mode bypassed the planner. Five phase-25 candidates: explicit live-vs-backtest Sharpe reconciliation, seed-stability test run + persistence, live → optimizer warmstart feedback, plateau-detection enforcement, CPCV refactor (Lopez de Prado canonical).

## Code-grounded findings

### F-1: Walk-forward correctness — standard expanding windows

`backend/backtest/backtest_engine.py` implements standard walk-forward optimization (WFO):
- Train 2010-2015 → Test 2016
- Train 2011-2016 → Test 2017
- ... etc.

Embargo logic prevents look-ahead bias (per Lopez de Prado AFML). Sub-period evaluation at `run_harness.py` further hardens with 3 sub-periods. **Gap vs canonical**: CPCV (Combinatorial Purged CV) generates a distribution of OOS results rather than one path — Lopez de Prado advocates CPCV as superior. pyfinagent uses standard WFO; this is a known but accepted compute-budget tradeoff.

### F-2: Seed stability — hardcoded but maybe untested

`backend/backtest/backtest_engine.py` uses `random_state=42` at 4 sites: lines 725, 749, 886, 914. Same params → same results (deterministic). 

`backend/api/backtest.py:1330` exposes `GET /api/backtest/harness/seed-stability` reading from `handoff/data/seed_stability_results.json`. **Critical gap**: this JSON file does NOT appear in `git status` — suggests either it doesn't exist, OR is gitignored, OR has not been refreshed. Cannot verify seed-stability empirically without running the test endpoint.

Phase-25 candidate `25.B6` addresses this with a forced run + commit of baseline results.

### F-3: Live-vs-backtest reconciliation — NAV-proxy only

`backend/services/reconciliation.py` (~200 LOC) implements a SHADOW backtest:
- Zero slippage, zero transaction cost
- Replays paper trades against yfinance adj-close prices
- Computes NAV divergence threshold: 5%

`backend/services/paper_go_live_gate.py:91-94`:

```python
# Use divergence as a proxy for Sharpe gap when explicit backtest Sharpe isn't available
# -- stays conservative.
if reconciliation.divergence_pct > 0.05:
    return False  # gate FAILED on SR-gap-by-proxy
```

**No code computes `live_realized_sharpe - backtest_predicted_sharpe` explicitly.** `backend/services/perf_metrics.py:84-106` `compute_sharpe_from_snapshots()` produces a paper-trading Sharpe independently, but it is never compared against the champion backtest's predicted Sharpe. The 30% SR-gap threshold mentioned in `paper_go_live_gate.py:38` is therefore enforced via NAV proxy, not explicit Sharpe comparison.

Industry benchmark per arxiv 2512.12924 (2025): 20-30% live performance decay is expected. pyfinagent's 5% NAV divergence + 30% SR gap thresholds are conservative — likely correct in principle but the proxy approach loses signal: a 5% NAV divergence with similar Sharpe trajectories is materially different from a 5% NAV divergence where Sharpe halved.

### F-4: MDA backtest → live (one-directional, working)

`backend/backtest/backtest_engine.py:59-75` persists MDA feature importances to `backend/backtest/experiments/mda_cache.json`. The live `quant_model` tool loads from this cache, so backtest-derived feature weights drive live signal generation. **This is the primary backtest → live channel.** Confirmed working.

`backend/backtest/experiments/optimizer_best.json` (38 lines) similarly persists champion params (current: Sharpe 1.1705, DSR 0.9526, saved 2026-04-06). The harness loads these as the starting point for the paper-trading engine.

**Gap (one-directional)**: there is NO live → backtest channel. Live paper-trading outcomes do not feed back into:
- Optimizer warmstart (next backtest run starts from `optimizer_best.json` parameters, not adjusted by live realized performance)
- Strategy selection (no live performance attribution per strategy)
- MDA re-computation (live signal generation doesn't update MDA weights based on which features actually helped)

This is the core "feedback loop" the hypothesis identified — confirmed absent.

### F-5: Optimizer plateau

`backend/backtest/experiments/quant_results.tsv` (60+ rows): most recent run `0083971f` ran 62 experiments on 2026-04-21, ALL DISCARDED. Baseline: Sharpe 1.1705, DSR 0.9526. No experiment outperformed baseline.

`scripts/harness/run_harness.py` planner has "Rule 1: PLATEAU after 10 consecutive discards → suggest strategy change." Should have fired at experiment 11 of run `0083971f`. The run continued to experiment 62 instead. Two hypotheses:
- (a) Run was launched in standalone optimizer mode bypassing the planner
- (b) Planner ran but its strategy-change suggestion was ignored / not acted on

Cross-link bucket 24.3 (autoresearch wiring) — the `meta_evolution/cron.py` weekly Sunday cron doesn't read `optimizer_best.json` and the proposer never runs scheduled. If plateau-detection fired in the planner but the strategy-switch was a soft suggestion without a strategy-rotation registry to write to, it would have been logged and ignored.

### F-6: 25-endpoint backtest API

`backend/api/backtest.py` (~1400 LOC) exposes 25 endpoints:
- POST `/api/backtest/run` (start backtest)
- GET `/api/backtest/runs` (list runs)
- GET `/api/backtest/runs/{run_id}` (status)
- GET `/api/backtest/runs/{run_id}/results` (full results)
- GET `/api/backtest/sub-period-results/{run_id}`
- GET `/api/backtest/optimizer-best` (current champion)
- POST `/api/backtest/optimizer/run` (start optimizer)
- GET `/api/backtest/harness/cycles` (harness log)
- GET `/api/backtest/harness/seed-stability` (F-2 anchor)
- ... 16 more endpoints for sprint state, experiment history, feature ablation, etc.

Surface is comprehensive. Operator-visible coverage is good.

### F-7: DSR / PBO gating

`scripts/harness/run_harness.py:114-126` `_count_experiments()` reads the full TSV row count — feeds DSR deflation properly per Lopez de Prado AFML 3rd Law. PBO gate at 0.20 (per autoresearch `gate.py:62`). Reality-gap at 2x costs Sharpe > 0.5.

These gates are correct and enforced. The harness logs CONDITIONAL when reality-gap divergence > 5% threshold (per Cycle 1 entry in `harness_log.md`).

### F-8: Standalone optimizer cycle observed

The most recent `harness_log.md` `Cycle 1 -- 2026-05-12 04:49 UTC` entry shows:

- Generator: 0 trials, Sharpe 0.0000 → 0.0000, kept=0

This is a dry-run harness invocation. The "real" optimizer plateau (F-5) was likely a standalone run NOT through this harness path.
