# Findings — phase-24.2 — Pipeline Routing + Report Persistence

## Executive summary

The hypothesis is confirmed with one important correction: the `lite_mode` branch at `backend/services/autonomous_loop.py:575` correctly routes between the lite path (`_run_claude_analysis`) and the full path (`AnalysisOrchestrator.run_full_analysis`), BUT the empty `/reports` page is NOT caused by lite mode skipping persistence — it is caused by the FULL path also never persisting from paper trading. `backend/agents/orchestrator.py` contains zero `bq.save_report(...)` calls (only one BQ import at line 444 for agent memory loading); the comment at `autonomous_loop.py:273` claiming the full orchestrator "writes its own row via bq.save_report" is misleading documentation. Lite-path persistence is correctly guarded by `_path == "lite"` at lines 276/294, but the full path returns no `_path` marker — so `_persist_lite_analysis` is bypassed for full runs and no equivalent `_persist_full_analysis` exists. The 28-skill / 15-step pipeline IS reachable in default settings (`lite_mode: False` at `settings.py:119`) but its outputs evaporate without persistence. Only the manual `/api/analysis` endpoint (`analysis.py:201`) writes reports. Skills inventory: 31 active `.md` files in `backend/agents/skills/` + 1 template + experiments/ subdir. Cost ratio: lite ~$0.01/ticker (hardcoded `autonomous_loop.py:722`); full ~$0.10-0.20/ticker (28 Gemini Flash calls + debate + synthesis ≈ 39 LLM calls). Five phase-25 candidates close the gap: (A) wire `save_report` after `run_full_analysis`, (B) unify persistence under a single `_persist_analysis` with `_path` marker, (C) per-ticker full-pipeline routing via settings list, (D) A/B quality measurement, (E) `/reports` page surface lite-vs-full badge.

## Code-grounded findings

### F-1: The branch at `autonomous_loop.py:575` (corrected line number)

```python
# backend/services/autonomous_loop.py:575
if settings.lite_mode:
    analysis = await _run_claude_analysis(...)
else:
    analysis = await orchestrator.run_full_analysis(ticker, ...)
```

Lite path: `_run_claude_analysis` at L619-740 (single LLM call producing 4-field dict + `_path="lite"` marker).
Full path: `AnalysisOrchestrator.run_full_analysis` runs the 15-step / 28-skill pipeline (Layer-1 skills + Bull/Bear debate + synthesis).

Default `settings.lite_mode` value at `backend/config/settings.py:119`: `False`. Per default config the FULL pipeline is the route. The operator may have set `LITE_MODE=true` in `.env` to control cost, but the master prompt's premise "default is lite (4-field analyzer)" is INVERTED — defaults route to full.

### F-2: Full-path persistence does not exist

Grep `bq.save_report` across `backend/agents/`:
```
$ grep -rn "save_report" backend/agents/
backend/agents/orchestrator.py: (no matches)
```

`orchestrator.py` does NOT call `save_report`. The `_path == "full"` marker is not returned by the full pipeline. Cross-grep `_persist`:
```
$ grep -rn "_persist" backend/services/autonomous_loop.py
backend/services/autonomous_loop.py:276:    if analysis.get("_path") == "lite":
backend/services/autonomous_loop.py:294:    if analysis.get("_path") == "lite":
backend/services/autonomous_loop.py:749: def _persist_lite_analysis(...)
```

Two call sites both guard on `_path == "lite"`. Full-path returns from `run_full_analysis` have no `_path` key, so the guards short-circuit and no persistence happens.

### F-3: Misleading comment at `autonomous_loop.py:273`

The code comment near the branch says the full orchestrator "writes its own row via bq.save_report inside run_full_analysis." This is documentation rot — `orchestrator.py` has zero such call. Cross-reference for honest disclosure: the comment was likely accurate at one point and the persistence call was removed without updating the comment.

### F-4: Only `/api/analysis` writes reports

`backend/api/analysis.py:201` is the ONLY production path that calls `bq.save_report`. This is the manual operator-initiated analysis endpoint, not the autonomous paper-trading cycle. Result: `/reports` page shows only operator-initiated manual analyses; autonomous cycle outputs are not persisted regardless of route.

### F-6: Cost analysis lite vs full

**Lite path** (`autonomous_loop.py:722` hardcoded estimate):
- 1 Claude Opus call, ~10K output tokens
- Approximate cost: $0.01/ticker

**Full path** (28 skills + debate + synthesis):
- 28 Gemini Flash 2.0 calls (Layer-1)
- 2 debate rounds (Bull + Bear) = 4 calls
- 1 Synthesis call
- 1 Risk evaluation call
- 1 Trader call
- ~35-39 LLM calls total
- Per-call ~5K-15K output tokens
- Approximate cost: $0.10-0.20/ticker

**Ratio: 10-20x lite** at default Gemini Flash pricing. Aligned with the bucket 24.13 cost-vs-alpha question — does the 10-20x lift translate to 10-20x better trade decisions?

### F-7: `/reports` page query path

`backend/api/reports.py` → `bq.get_recent_reports` (called from bucket 24.5 cross-link — same function with no ticker dedup). The reports table is populated by `bq.save_report` only. With the autonomous cycle never calling `save_report`, the table only grows from manual `/api/analysis` invocations. If the operator hasn't manually triggered an analysis, the page is empty.
