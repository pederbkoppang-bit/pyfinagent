# Harness Stress-Test -- Opus 4.7 (Q2 2026)

**Phase:** phase-40.3 (OPEN-26)
**Date:** 2026-05-23
**Cycle:** 49
**Model:** Claude Opus 4.7 (claude-opus-4-7[1m])
**Doctrine:** [Anthropic Harness Design for Long-Running Apps](https://www.anthropic.com/engineering/harness-design-long-running-apps)

> "Every component in a harness encodes an assumption about what the model can't do on its own, and those assumptions are worth stress testing. On each new Claude model release, re-run a representative step WITHOUT the harness and compare the output to the harness-produced result. If the model now does X on its own, remove the scaffolding for X."

Opus 4.7 was released 2026-04-16; we have 5+ weeks of operating experience. This is the first formal stress-test cycle.

---

## 1. Methodology (honest framing)

We cannot literally re-execute a past step in a parallel session. Instead we reconstruct from `handoff/harness_log.md` what the model did *with* the harness across cycles 12-47, and reason carefully about what it would have done *without* the harness components -- using:

- The actual harness output (preserved in handoff/archive/phase-*/)
- The cycle-2 corrections that fired (empirical evidence of Q/A value)
- The researcher findings that contradicted the masterplan (empirical evidence of researcher external value)
- Opus 4.7 release-notes capability claims (https://platform.claude.com/docs/en/about-claude/models/whats-new-claude-4-7)

## 2. Steps picked for analysis

### 2.1 Primary: phase-37.3 NO_OP closure (Cycle 46, 2026-05-23)

**Step:** `budget_tokens deprecation cleanup (OPEN-18)`. Masterplan claimed: "orchestrator.py:99-117 + debate.py:63 still use deprecated budget_tokens; should be `thinking_budget` per Vertex AI 2026 SDK."

**Harness-produced outcome:**
- Researcher (5 sources read-in-full, tier=simple) determined the audit_basis was FACTUALLY WRONG: `budget_tokens` is **Anthropic's** wire-literal field name for legacy models (Opus 4.5 and older); deleted -- not renamed -- in Opus 4.7+. `thinking_budget` is **Gemini's** field name; already correctly used at `llm_client.py:917` via `ThinkingConfig(thinking_budget=...)`.
- Main wrote contract with HONEST DUAL-INTERPRETATION pattern: 3 PASS tests (operational equivalents) + 1 xfail strict (literal criterion unsatisfiable without breaking Anthropic legacy support). Added phase-37.3.1 (P3) follow-up: "Re-evaluate when Anthropic legacy-model EOL announced."
- Q/A round-1 PASS. Top-15 code-review heuristic sweep: 0 BLOCK / 0 WARN / 0 NOTE.
- ZERO production-code lines changed.

**Counterfactual: what would Opus 4.7 do harness-free?**
1. Read masterplan 37.3 verification: `test $(grep -rn 'budget_tokens' backend/ --include='*.py' | wc -l) -eq 0`. Find 45 refs.
2. Without researcher: Opus 4.7 has the relevant API knowledge in its training (post-Feb 2025 Anthropic docs cover both `budget_tokens` and the new `effort` param). It would likely identify the conflation.
3. Without Q/A: Opus 4.7 might still write the dual-interpretation pattern -- it's in CLAUDE.md as a documented honest path.
4. Without handoff files: Opus 4.7's 1M context can hold the full masterplan + relevant source. No state loss for a single cycle.

**Gap analysis:**
| Component | Harness value | Harness-free likely outcome |
|---|---|---|
| Researcher external research | 5 sources cited in contract; provides paper-trail | Opus 4.7 could write the same brief from training but WITHOUT verifiable citations |
| Researcher tier-knob | tier=simple; sets 5+ source floor | Without floor, depth is non-deterministic per-cycle |
| Q/A independent verification | Cross-checked llm_client.py:1388 + :917 | Opus 4.7 self-evaluating WOULD likely PASS but Anthropic explicitly warns: "agents tend to confidently praise their own work" |
| handoff/contract.md | Documents N* delta + scope honestly | Doable in commit message OR /goal-tracking issue tracker |
| handoff/evaluator_critique.md | Independent record of Q/A reasoning | Lost without Q/A subagent (since Main shouldn't self-evaluate) |

**Verdict:** For SIMPLE NO_OP closures like phase-37.3, ~70% of the scaffolding ROI is in the *citation paper-trail* (researcher) and the *independent verification* (Q/A). The contract + handoff files add ~20%. The full ritual adds ~10% friction. NET POSITIVE for NO_OP closures because the cost is small (~15 min/cycle).

### 2.2 Secondary (counter-example): phase-38.6.1 cycle_lock wiring (Cycle 44, 2026-05-23)

**Step:** `Wire cycle_lock into autonomous_loop + main.py lifespan (depends 38.6)`.

**Harness-produced outcome (with cycle-2 save):**
- Main SKIPPED researcher with rationale ("cycle 43 brief Section C is the literal wiring sketch; this is literal execution of prior research; no new domain").
- Q/A round-1 **CONDITIONAL** on TWO blockers: (a) researcher SKIPPED -- operator memory `feedback_never_skip_researcher` (2026-05-22) overrode this carve-out; (b) `experiment_results.md` was STALE phase-38.5 content not refreshed for phase-38.6.1 -- five-file protocol breach.
- Cycle-2 recovery: retroactive researcher spawn (gate_passed=true, 7 sources, work confirmed SOUND); experiment_results.md rewritten for phase-38.6.1.
- Q/A round-2 fresh spawn on UPDATED evidence: PASS.

**Counterfactual: what would Opus 4.7 do harness-free?**
1. Without Q/A: Main self-evaluating would have flipped status to done with the SKIP-disclosed-openly and the stale experiment_results untouched. Anthropic's explicit warning ("agents tend to confidently praise their own work") + the cycle-46 retrospective ("Main systematically under-estimates researcher value on wiring-only steps") suggest this is the EXACT failure mode the harness exists to catch.
2. Without researcher: the wiring would have been written based on cycle-43 brief alone -- which IS sufficient (researcher round-2 confirmed SOUND). BUT the operator memory exists precisely because Main slipped on 7 of 9 phase-4.8 cycles. The researcher floor is a PROCESS DISCIPLINE not a CAPABILITY GAP.
3. Without handoff files: the cycle-2 flow would be impossible. The "fresh Q/A on UPDATED evidence" pattern requires durable file-based handoff -- Anthropic's documented pattern.

**Verdict:** Harness saved this cycle. Without Q/A, the protocol breach goes undetected. Without handoff files, the cycle-2 fix cycle is impossible. SCAFFOLDING IS LOAD-BEARING here.

## 4. Opus 4.7 capability deltas vs Opus 4.5

Per https://platform.claude.com/docs/en/about-claude/models/whats-new-claude-4-7:

- **Adaptive extended thinking + effort parameter** -- replaces `budget_tokens` (related: this stress-test's primary step picked phase-37.3).
- **1M context window** -- mitigates GitHub issue #51060 (subagent context loss). Operator already takes advantage via Max-subscription Opus 1M (per CLAUDE.md effort policy).
- **GPQA Diamond / GDPval-AA improvements** -- 17pt / 79-Elo gap over Sonnet 4.6 (CLAUDE.md cites phase-29.2 audit). Quality-depth on research-synthesis is where Opus 4.7 shines; the researcher subagent benefits MORE from Opus 4.7 than the Main session does.

**Implication for harness design:** Opus 4.7 INCREASES the marginal value of Researcher + Q/A (better synthesis, better audit-quality) without obviously decreasing Main's need for them.

## 6. Anti-pruning: what we CONFIRMED must stay

After this stress test, the following are MORE clearly load-bearing than expected:

1. **Q/A subagent** -- cycle-44 in-session save was direct empirical evidence. Anthropic's anti-self-eval guidance is explicit. No model improvement removes this need; it's a structural separation-of-duties principle.
2. **Researcher external-research half** -- cycle-46 caught a factually wrong audit_basis. Operator memory exists because Main slipped on 7 of 9 cycles.
3. **Cycle-2 file-based fresh-respawn pattern** -- not a "scaffold" so much as a documented design pattern. Anthropic's harness-design + multi-agent-research-system blog posts BOTH describe this.
4. **Live_check gate** (phase-23.8.1 / R-1) -- converts "agent claimed PASS" into "operator-auditable artifact". The dev-MAS audit basis for this is exactly the failure mode the harness exists to catch.
